OSI (Open Systems Interconnection):

• El modelo OSI es una conceptualización que divide la comunicación en redes en siete capas (física, enlace de datos, red, transporte, sesión, presentación y aplicación). Cada capa tiene una función específica y ayuda a estandarizar la comunicación entre diferentes sistemas de red.

IP (Internet Protocol):

• El Protocolo de Internet (IP) es el principal protocolo de la capa de red que direcciona y entrega paquetes de datos desde el origen hasta el destino. Las direcciones IP identifican de manera única cada dispositivo en una red.

TCP (Transmission Control Protocol):

El Protocolo de Control de Transmisión (TCP) es un protocolo orientado a la conexión que garantiza la entrega segura y ordenada de un flujo de datos entre dos dispositivos. TCP realiza comprobaciones de errores y retransmite paquetes perdidos.

UDP (User Datagram Protocol):

UDP (User Datagram Protocol): • El Protocolo de Datagrama de Usuario (UDP) es un protocolo de comunicación sin conexión que permite el envío de datagramas sin asegurarse de su entrega. Es rápido y eficiente, pero no garantiza la llegada de los datos ni el orden en que se reciben.

SSH (Secure Shell):

• SSH es un protocolo de red criptográfico utilizado para realizar operaciones remotas seguras, como iniciar sesión en servidores y ejecutar comandos. Utiliza cifrado para proteger la comunicación entre el cliente y el servidor.

FTP (File Transfer Protocol):

• El Protocolo de Transferencia de Archivos (FTP) es un protocolo estándar que permite la transferencia de archivos entre un cliente y un servidor a través de una red. FTP puede usar autenticación y puede operar en modo activo o pasivo.

SMTP (Simple Mail Transfer Protocol):

• El Protocolo Simple de Transferencia de Correo (SMTP) es el protocolo utilizado para enviar correos electrónicos a través de Internet. Los servidores SMTP reciben, encolan y retransmiten correos electrónicos a otros servidores.

DHCP (Dynamic Host Configuration Protocol):

• El Protocolo de Configuración Dinámica de Host (DHCP) asigna automáticamente direcciones IP a los dispositivos en una red, lo que facilita la gestión de direcciones IP y reduce errores manuales.

DNS (Domain Name System):

• El Sistema de Nombres de Dominio (DNS) traduce nombres de dominio legibles por humanos (como www.ejemplo.com) en direcciones IP numéricas que las computadoras utilizan para identificar y comunicarse entre sí en la red.

HTTP (Hypertext Transfer Protocol):

• El Protocolo de Transferencia de Hipertexto (HTTP) es el protocolo base para la comunicación en la web. Permite la transferencia de recursos como páginas web, imágenes y videos desde un servidor web al navegador del usuario.

ARP (Address Resolution Protocol):

• El Protocolo de Resolución de Direcciones (ARP) es utilizado para mapear direcciones IP a direcciones MAC (Media Access Control) en una red de área local. ARP permite que los dispositivos encuentren la dirección física de otro dispositivo en la misma red.

MAC (Media Access Control):

• La dirección MAC es una dirección física única asignada a la tarjeta de red de cada dispositivo. Es utilizada para identificar dispositivos en una red de área local y asegurar la correcta entrega de paquetes de datos.

LAN (Local Area Network):

• Una Red de Área Local (LAN) es una red que conecta dispositivos dentro de un área geográfica limitada, como una oficina, una escuela o una casa. LAN permite la comunicación y el intercambio de recursos entre dispositivos conectados.
